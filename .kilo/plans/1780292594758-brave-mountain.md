# Fix Route Details Edit Functionality

## Requirements (Confirmed)
- **Vehicle Type**: Read-only (cannot change from Bus/Train/Plane/private)
- **Transport Mode**: Read-only
- **Editable Fields**: 
  - For all routes: departure time/date, arrival time/date, vehicle number
  - For **Plane routes**: airline name, flight number, baggage type, max weight
  - For **Train routes**: train number/name, class type, PNR number, seat numbers, max weight
  - For **Bus routes**: service name, bus number, max weight

## Implementation Plan

### 1. VehicleDetailsCard.jsx Changes
- Remove Vehicle Type dropdown from edit mode (show as read-only)
- Show vehicle type as read-only text with appropriate icon
- Add editable input fields for transit details:
  - **Plane**: airline_name input, flight_number input, baggage_type select
  - **Train**: train_number input, train_name input, class_type select
  - **Bus**: service_name select/input, bus_number input
- Hide Vehicle Number field for public transport (not applicable)

### 2. RouteDetails.jsx Changes (PARCEL PREFERENCES FIX)
- **Line 449**: Fix case mismatch in auto-selection:
  ```javascript
  // BEFORE (broken):
  const selected = d.parcelPreferences.acceptedTypes.includes(type);
  
  // AFTER (fixed):
  const selected = d.parcelPreferences.acceptedTypes.map(t => t.toLowerCase()).includes(type.toLowerCase());
  ```
- This ensures `"Documents"` matches `"documents"` in the array

### 3. JSON Structure for Updates
- Will send `transit_details` object with updated fields
- API endpoint at `travellerRoute.service.js` supports updating transit_details via `updateTravellerRoute`
